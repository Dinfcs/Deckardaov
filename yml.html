Conceptos Básicos de YAML

¿Qué es YAML?

YAML (YAML Ain’t Markup Language) es un lenguaje de serialización de datos, legible por humanos, usado para definir configuraciones, estructuras de datos, y más.

1. Clave-Valor

name: "Luis"
age: 30

2. Objetos (Mappings)

author:
  name: "Luis"
  email: "luis@example.com"

3. Listas (Sequences)

languages:
  - Python
  - JavaScript
  - Bash

4. Lista de Objetos

servers:
  - name: "web01"
    ip: "192.168.1.10"
  - name: "db01"
    ip: "192.168.1.20"

5. Indentación

Usa espacios (no tabuladores) para definir jerarquías.

6. Comentarios

# Este es un comentario
name: "Luis"  # Comentario al final de línea

7. Tipos de Datos

TipoEjemplo
Cadena"Hola mundo"
Número42, 3.14
Booleanotrue, false
Nulonull, ~
Lista- item1, - item2
Objetoclave: valor

8. Cadenas Multilínea

description: |
  Este es un texto
  que ocupa varias líneas

9. Alias y Referencias

defaults: &common
  theme: "dark"
  lang: "es"

user1:
  <<: *common
  name: "Luis"

10. Valores Vacíos y Nulos

empty_value: ""
null_value: null
another_null: ~

11. Estilos de Claves

name: Luis
"full name": Luis
"123": valor
"true": valor

12. Errores Comunes

  • Usar tabuladores en lugar de espacios
  • Mezclar tipos de indentación
  • No cerrar comillas
  • Usar claves duplicadas en el mismo nivel

Ejemplo Completo

author:
  name: "Luis"
  email: "luis@example.com"
  social_networks:
    - type: "GitHub"
      url: "https://github.com/luis"
    - type: "LinkedIn"
      url: "https://linkedin.com/in/luis"
languages:
  - Python
  - JavaScript
  - Bash
active: true
description: >
  Este es un archivo YAML
  que muestra múltiples conceptos.

Resumen de lo que debes dominar

  • Leer y escribir estructuras clave-valor
  • Crear listas y objetos anidados
  • Usar correctamente la indentación
  • Entender los tipos de datos
  • Comentar y documentar archivos
  • Validar sintaxis y evitar errores comunes
  • Reutilizar bloques con alias si es necesario
